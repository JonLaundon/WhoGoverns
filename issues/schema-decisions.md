# Schema decisions log

- **2026-07-05 — Spiral 1 active schema set.** Annex A13.1 lists the "five core schemas (Body, Source, Power, Duty, Veto)". The Spiral 1 POC populates Body, Source, Office, PersonRole, Relationship, Budget and Staffing, but **not** Power/Duty/Veto (deferred to Spiral 2). So the active/validated set for Spiral 1 is those seven; Power/Duty/Veto ship as draft stubs and are not validated against. Reasoned deviation from the literal A13.1 list, which predates the v0.3 officeholder/finance POC scope. Flag for sponsor review.
- **2026-07-05 — `external_ids` object added to Body schema.** To store the GOV.UK `content_id` and `analytics_identifier` (stable external IDs, A5.1 intent) without polluting the top-level record. Holds govuk_content_id, govuk_analytics_identifier, company_number, charity_number.
- **2026-07-05 — `status` vs `govuk_status`.** GOV.UK `govuk_status` values live/exempt both map to body `status = active` (exempt = exempt from GOV.UK publishing, not inactive). closed -> closed.
- **2026-07-06 — Body schema: `needs_classification_review` (boolean, optional) added.** Task 2 sets it `true` where the API `format` is coarse/coerced (the map's `flag_for_review` list: Sub organisation, Court, Other, and the singletons) or unmapped. A queryable flag beats parsing notes; drives the map and the classification-refinement pass. 338/663 flagged. Additive; seeds omit it (absent = not flagged).
- **2026-07-06 — Body schema: `status` gains `forming`.** `govuk_status` joining/transitioning → `status: forming` (raw value in notes), so in-formation bodies (11) are included and rendered with a dashed outline (sponsor decision, 2026-07-06). Enum is now active|forming|closed|merged|superseded.
- **2026-07-05 — RESOLVED (was OPEN): `govuk_format_to_body_type` map had stale/missing keys.** The full ingest surfaced live `format` strings the v0.1 map mis-keyed or omitted (bare `"Tribunal"` not `"Tribunal non-departmental public body"`; `"Devolved government"` not `"Devolved administration"`; plus five unmapped formats). Superseded by the taxonomy decision below (2026-07-06). Map rewritten to v0.2 covering all 16 formats present in the dataset.
- **2026-07-06 — DECISION (sponsor, Jon): body_type taxonomy re-based on the machineryofgovernment.uk category scheme (pattern only, no data reuse).** `body_type` vocab → v0.2; body schema enum updated; format map → v0.2. Changes vs v0.1:
  - **Dropped `regulator`.** Regulation is a *function* (captured via powers, Spiral 2), not a structural type. Regulators keep their statutory form — Ofgem was already seeded `non_ministerial_department`, so no seed churn.
  - **Renamed** `non_departmental_public_body` → `executive_ndpb`; `advisory_body` → `advisory_ndpb`; `other` → `other_body`. Reseeded the one affected record (ACOBA → `advisory_ndpb`).
  - **Added `division_directorate`** — target for the API `"Sub organisation"` format (126 live). Heterogeneous; review-flagged.
  - **Added `royal_charter_body`**, with a hard caveat: chartered bodies (Law Society of Scotland, ICAEW, RCVS, CWGC, Royal Hospital Chelsea, etc.) are **absent from the GOV.UK Organisations API** (verified: 0 of 8 sampled present). They are self-governing bodies incorporated via the Privy Council, deliberately outside the government org register. So `royal_charter_body` is **NOT populated from the API** — it is a separate primary-sourced tranche from the Privy Council chartered-bodies register, each record cited to its charter (A2.2), sequenced like the budgets/staffing bolt-on. No Spiral 1 API body auto-classifies to it. Open scope question for that tranche: several chartered bodies are professional self-regulators, not government bodies — decide inclusion when building it.
  - **Courts** (24 live) → `other_body` + review flag (constitutionally distinct from tribunals; possible dedicated `court` class later).
  - Rule retained (AGENT.md rule 3): any unmapped `format` → `other_body` + review flag, never dropped. Coarse/coerced formats also carry a review marker (see `flag_for_review` in the map).
  - **Decision register (plan §10):** logged as new decision **#12** (Closed 2026-07-06). (Note: register #10 is the *Tranche A0 body list*, a different item — not this taxonomy; an earlier draft of this note misread it.)
- **2026-07-06 — DECISION (sponsor, Jon): officials/offices are first-class, power-holding nodes — NOT a body_type.** Rationale: most UK statutory powers vest in an **office** ("the Secretary of State may…", "the Prime Minister shall…", "the Treasury may…"), which in law is a **corporation sole** — a continuous legal person distinct from both the human holder and the hosting department. Modelling consequences:
  - **Do not** add PM/ministers to `body_type` (that would collapse people into machines). They are the **Office / PersonRole** layer (already first-class in A4; leave draft in Spiral 1 per the A4.206 exception).
  - **Annex A6 revision needed (flag for sponsor):** the draft Power/Duty/Veto records bind to `body_id` only. They must gain a **polymorphic `holder`** — `office_id` OR `body_id` — so office-held powers are expressible without mis-attributing them to a department. (The annex's own Duty example already reads `"trigger": "When the Secretary of State proposes to…"`, i.e. the actor is the office.) Build in **Spiral 2** when Power/Duty/Veto leave draft; recorded now so officials cannot be quietly skipped.
  - **Spiral 1 presentation:** the map is **office-centred** — PM's office → Cabinet offices → other ministerial offices → departments → agencies/ALBs — with Office as a node, not a department attribute. See the Spiral 1 workpack note (tasks 4 and 6).
